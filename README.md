## Overview

This repository contains a **Python based Recruitment Management System (RMS)** developed to support **governed, secure and transparent recruitment workflows** aligned with club standards for **data governance and operational security**.

The platform centralizes candidate data management, interview evaluation, reviewer validation and secure report generation with strong emphasis on **presentation and accountability**.

---

## Objectives

- Enable structured and policycaligned recruitment operations  
- Ensure secure handling of candidate and evaluator data  
- Enforce role based governance and access control  
- Generate officially formatted reports  

---

## Key Features

### Dashboard & Recruitment Analytics
- Total number of candidates registered  
- Number of candidates attended interviews  
- Number of candidates pending for interview  
- Interview completion status  
- Reviewer evaluation status  
- Real time recruitment progress monitoring  

---

### Candidate Data Management
- Bulk import of candidate details using a standardized CSV format  
- Automated validation of imported records  
- Centralized candidate profile repository  
- End to end candidate status tracking  

---

### Interview Management
- Verification of candidate information  
- Structured and documented interview comments  
- Interview status updates  
- Interviewer identity automatically fetched from authenticated user accounts  

---

### Reviewer Evaluation Workflow
- Independent reviewer access  
- Validation of interviewer feedback  
- Reviewer comments recorded for governance and quality assurance  
- Clear separation of interviewer and reviewer responsibilities  

---

### Checklist-Based Final Evaluation
- Standardized checklist driven evaluation model  
- Ensures consistency, fairness and transparency  
- Supports documented and reviewable final decisions  

---

### Secure & Official Report Generation
- Automatic generation of **A4-size official interview reports**
- Reports follow **documentation standards**
- Formatting includes:
  - **Tahoma font** for formal and official appearance  
  - GDG OnCampus watermarking  
  - Security hardened, tamper resistant layout  

---

### Security, Governance & Audit Logging
- Displays last login details for users  
- Captures and logs:
  - IP address  
  - Location  
  - ISP details  
- Enables compliance oriented audit trails  

---

### Role Based Access Control (RBAC)
- Administrator, Interviewer and Reviewer roles  
- Role based permission enforcement  
- Protection of sensitive recruitment data  

---

## Technology Stack

| Layer | Technology | Purpose |
|------|-----------|--------|
| Backend | Python | Core business logic |
| Web Framework | Flask / Django | Application routing and APIs |
| Database | SQLite / PostgreSQL / MySQL | Secure data storage |
| Frontend | HTML, CSS, JavaScript | UI and dashboards |
| Reporting | ReportLab / WeasyPrint | A4 PDF reports |
| Security | RBAC & Audit Logs | Governance |
| Version Control | Git & GitHub | Source control |

---

## Governance & Compliance Focus

The system emphasizes:
- Data confidentiality and integrity  
- GDGOnC-KARE alignment  
- Role based accountability  
- Official documentation standards  

---

## Project Scope

This platform is purpose built for **GDGOnC - KARE community recruitment workflows** and is not intended for commercial hiring without further compliance extensions.
